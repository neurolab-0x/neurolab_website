# Contributing to Neurolab Website

Thanks for your interest in contributing! We welcome improvements, bug fixes, and documentation updates.

How to contribute:
1. Fork the repository (or branch from `neurolab-0x/neurolab_website` if you have push access).
2. Create a feature branch:
   ```bash
   git checkout -b feature/my-change
   ```
3. Make changes with small, focused commits.
4. Run tests and linters locally before pushing:
   ```bash
   npm run test
   npm run lint
   ```
5. Push your branch and open a Pull Request against `neurolab-0x:main`.
6. In the PR description, explain the change, list any relevant issues, and include screenshots or logs if applicable.

Guidelines:
- Follow the existing TypeScript style and patterns.
- Write or update tests for bug fixes and new features.
- Keep changes backwards-compatible where possible.
- Break large changes into multiple PRs.

Reporting issues:
- Open an issue with a clear title, steps to reproduce, and expected vs actual behavior.

Maintainers will review PRs and may request changes. Thank you for helping improve Neurolab!
